HASIL PROGRAM DARTPAD - CONSTRUCTOR & ENKAPSULASI

1. Kelas Buku

Pengertian: Kelas Buku memodelkan data buku dengan atribut seperti judul, penulis, dan harga. Konstruktor parameterized digunakan untuk menginisialisasi objek saat dibuat.

2. Kelas Mobil

Pengertian: Kelas Mobil memodelkan data mobil, dengan atribut merk, tipe, dan tahun. Parameterized constructor dan named constructor (baru() & dariJson()) digunakan untuk fleksibilitas pembuatan objek.

3. Kelas Warna

Pengertian: Kelas Warna memodelkan nilai RGB. Constant constructor digunakan agar objek bersifat immutable sehingga nilainya tidak bisa diubah setelah dibuat.

4. Kelas Titik

Pengertian: Kelas Titik memodelkan koordinat (x, y). Constructor dengan initializer list digunakan untuk memastikan nilai negatif diubah menjadi positif saat objek dibuat.

5. Kelas Mahasiswa

Pengertian: Kelas Mahasiswa memodelkan atribut mahasiswa seperti nama, NIM, dan jurusan. Terdapat default constructor dan named constructor (dariMap() & tanpaNama()) untuk fleksibilitas pembuatan objek.

6. Kelas Person

Pengertian: Kelas Person memodelkan individu dengan atribut nama dan umur. Getter dan setter digunakan untuk enkapsulasi sehingga data dapat diakses dan dimodifikasi dengan aman.

7. Kelas Produk

Pengertian: Kelas Produk memodelkan barang dengan atribut nama dan harga. Setter dilengkapi validasi agar data sesuai aturan, sehingga tidak ada harga negatif atau nama kosong.

8. Kelas User

Pengertian: Kelas User memodelkan akun dengan atribut username dan password. Setter dan method login serta ganti password digunakan untuk validasi dan keamanan data.

9. Kelas RekeningBank

Pengertian: Kelas RekeningBank memodelkan rekening dengan atribut saldo. Method deposit dan tarik tunai dilengkapi validasi agar transaksi aman dan saldo tidak negatif.

Kesimpulan

Melalui semua proyek DartPad di atas, dapat disimpulkan bahwa penggunaan constructor (parameterized, default, named, constant) dan enkapsulasi (getter dan setter) sangat penting dalam pemrograman Dart untuk memastikan objek dibuat dengan benar, data aman, dan perilaku program sesuai yang diharapkan. Dengan memahami konsep ini, programmer dapat membuat kode yang lebih terstruktur, modular, dan mudah dipelihara.
